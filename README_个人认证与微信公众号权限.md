# 个人认证与微信公众号权限说明

## 微信公众号认证类型

根据项目设计文档和微信平台规则，微信公众号分为以下几种类型：

### 1. 订阅号
- **个人订阅号**：个人可以申请，但功能权限有限
- **企业/组织订阅号**：需要提供相应资质，功能相对丰富

### 2. 服务号
- 通常为企业或组织申请
- 拥有更丰富的接口权限
- 需要更严格的资质审核

## 认证状态对权限的影响

### 未认证公众号
- 权限非常有限
- 无法使用大部分高级接口（如文章发布、素材管理等）
- 每月群发消息次数受限

### 已认证公众号
- 可以申请使用高级接口
- 拥有完整的发布能力
- 可以使用自定义菜单、微信支付等高级功能

## 个人认证的限制

根据项目设计文档和微信平台规则，个人认证存在以下限制：

1. **个人订阅号即使认证后也可能没有完整的发布能力**
2. **大部分高级接口需要企业认证的服务号才能使用**
3. **文章发布、草稿管理等功能通常要求公众号完成微信认证**

## 项目中的认证配置

项目中通过 `wechat_config.ini` 文件配置认证信息：

```ini
[wechat]
# 微信公众号AppID
app_id = wxa7338d443a01ceb5

# 微信公众号AppSecret
app_secret = afc83ad5e0ff5253b06be17ffabc3f39
```

## 解决方案建议

如果遇到 48001 (api unauthorized) 错误，可以考虑以下解决方案：

1. **检查公众号认证状态**：
   - 登录微信公众平台确认账号是否已完成认证
   - 查看账号类型（个人订阅号/企业服务号）

2. **申请接口权限**：
   - 在"开发"->"接口权限"中查看和申请相关权限
   - 确认是否拥有"发布能力"等相关权限

3. **升级账号类型**：
   - 如果是个人订阅号，考虑升级为企业认证服务号
   - 完成微信认证流程

4. **使用测试号**：
   - 开发阶段可使用微信提供的测试号进行功能验证
   - 测试号申请地址：https://developers.weixin.qq.com/sandbox

## 注意事项

- AppID 和 AppSecret 是敏感信息，不应硬编码在代码中
- 项目采用配置文件优先的方式管理认证信息，符合用户偏好
- 生产环境建议使用已认证的企业服务号
